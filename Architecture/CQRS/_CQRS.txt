CQRS - Command and Query Responsibility Segregation
https://habr.com/ru/company/simbirsoft/blog/329970/

Архитектурный принцип, который используется при масштабировании серверов сервиса. Когда их начинает быть больше одного.
Классическая многослойная архитектура не даёт лёгкого решения подобной проблемы. Одним из подходов, который сильно поможет в начале является CQRS.
Суть:
Разделение меняющего состояние кода, от того, который просто читает.

Разделение может быть логическим или физическим, пофиг.
В основе принцип CQS (Command-query separation), который говорит разделять методы классов на:
1) Queries - методы, которые возвращают какой-то результат ничего при этом не меняя
2) Commands - методы изменяют объект не возвращая результат

Основываясь на разделении логики чтения и записи можно очень хорошо распараллеливать операции Query.

CQRS идёт чуть дальше. Если CQS на уровне методов разделяет логику, то CQRS - на уровне объектов.
Для записи классы Command, для выборки Query.
