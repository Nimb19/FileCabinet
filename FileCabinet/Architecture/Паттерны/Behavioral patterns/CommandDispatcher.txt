# Чем отличается Диспетчер команд от Посредника:
https://stackoverflow.com/questions/57963474/what-are-the-differences-between-the-command-dispatcher-and-mediator-design-patt

В отличии от паттерна 'Посредник', не знает тонкостей работы, оркестровки, взаимодействия с компонентами, это тупой диспетчер, 
который берёт сообщение/нотификацию и просто ищет того кто принимает этот запрос.

Для чего:
Для DI, в частном случае используется в контроллерах к примеру, где мы не очень хотим каждый раз создавать контроллер с кучей зависимостей 
для обработки какого то запроса, который может использовать только одну из них.
Банально для упрощения кода в контроллерах. Просто избавляемся от лишних зависимостей и кода, который заключаем в класс обработчика.
