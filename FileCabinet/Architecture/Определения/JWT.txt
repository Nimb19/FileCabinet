JWT - Json Web Token
Используется в Web для хранения в куках на стороне клиента сессии к веб-приложению и информации о ней.

Самый очевидный кейс: хранит 
1) Сессию аутентификации
2) Доступные права

Ну и что хранит обязательно: 
1) Тип токена
2) Название алгоритма его шифрования
3) Сигнатура (хэш: Созданный на основе всех других хранимых данных + Секретного ключа с веб-сервера авторизации или публичного ключа на бэкенде)

За счёт чего токен может бэкендом верифицироваться (передаваться на сервер авторизации для проверки), 
что означает что, если он успешно верифицирован, то его никто не менял и мы ему полностью доверяем.
Поэтому один раз отдаём такую Json модельку клиенту -> дальше просто верифицируем в Middleware его, если на конкретный запрос это было нужно.


Технически информация выше хранится в спец. формате:
{
	"Header": { 
		"алгоритм шифрования токена" 
		"тип токена", 
	},
	"Payload": { 
		полезная нагрузка, сами решаем что нам нужно 
	},
	"Signature": "строка с хэш ключём"
}

Пример:
